# 北京空气质量数据挖掘分析报告

## 教学案例：Associate Rule Mining on Air Quality Data

**项目时间**: 2024年  
**数据集**: 北京空气质量监测数据 (2017年全年)  
**分析方法**: 关联规则挖掘 (Apriori 算法)  
**编程语言**: Python

---

## 摘要 (Abstract)

本报告对2017年北京地区全年空气质量监测数据进行了系统的数据挖掘分析。通过数据预处理、统计分析和关联规则挖掘，我们发现了污染物浓度与空气质量等级之间的强关联模式，为空气质量治理提供了数据支撑。

**主要发现**:
- 数据清洗前共1812记录，清洗后1790记录，去重率1.2%
- 发现22987条强关联规则（Lift > 1.5）
- PM2.5、PM10与NO₂间存在强相关性（相关系数>0.7）
- 冬季污染物浓度明显高于其他季节

## 四、关联规则挖掘 (Association Rule Mining)

### 4.1 算法选择与参数设置

**选择Apriori算法的理由**:
1. 传统、稳定、易于解释
2. 数据项数量可管理(35项)
3. 规则可读性强

**参数设置依据**:
- **最小支持度 (Min Support)**: 5%
  - 保证规则涉及≥90个记录
  - 过高(10%)会导致规则过少
  - 过低(1%)产生噪声规则
  
- **最小置信度 (Min Confidence)**: 50%
  - 规则预测准确度≥50%
  - 低于此值规则缺乏实用意义

- **最小提升度 (Min Lift)**: 1.5
  - 规则比随机预测好50%
  - 筛选出具统计显著性的规则

### 4.2 挖掘结果

| 指标 | 数值 |
|-----|------|
| 频繁项集数 | 3,841 |
| 候选规则数 | 26,906 |
| 强规则数 (Lift>1.5) | 22,987 |
| 强规则占比 | 85.4% |

### 4.3 强关联规则示例

#### 规则组1: 周末特定污染模式

```
规则: No2=高, 休息日, PM10=高 → So2=低, Co=低, "是"
支持度: 0.0631 (113天同时满足)
置信度: 0.7338 (73.4%的该条件日符合结论)
提升度: 11.62 (比随机概率高11倍)

解释: 
- 周末(休息日)：工业停产、汽车减少
- NO2高(汽车尾气): 城市主干道污染源
- PM10高: 建筑施工继续
- →SO2低、CO低: 工业排放减少
- 提升度高: 周末工业停产效果明显
```

#### 规则组2: 季节性污染关联

```
规则: 空气质量=轻度污染, 休息日 → No2=高, PM10=高, "是"
支持度: 0.0508 (91天)
置信度: 1.0000 (100%准确)
提升度: 11.62

解释:
- 当出现轻度污染且是休息日时
- 必然伴随NO2高+PM10高
- 反映周末特定污染模式的稳定性
```

#### 规则组3: 冬季污染模式

```
规则: PM2.5=高, 冬 → PM10=高, No2=高, SO2=高
支持度: 0.0842
置信度: 0.9156
提升度: 8.34

解释:
- 冬季PM2.5高时，其他主要污染物也高
- 反映冬季污染的多污染物叠加特征
- 与供暖期、逆温层、北风减少有关
```

### 4.4 规则解释与可视化

**频繁模式分类**:

1. **污染物组合模式** (65%)
   - {PM2.5=高, NO₂=高} → 交通污染
   - {PM10=高, SO₂=高} → 工业污染
   - {O₃=高, NO₂=低} → 光化学污染

2. **时间模式** (25%)
   - {冬季, 周末} → 低流动污染
   - {夏季, 法定节假日} → 清洁日
   - {工作日} → 高污染

3. **质量等级关联** (10%)
   - {PM2.5=高, NO₂=高} → 严重污染
   - {O₃=高, PM2.5=低} → 轻度污染

## 六、技术总结

### 6.1 数据挖掘工作流

```
原始数据 (1812行)
    ↓
[数据预处理]
  ├─ 去重: -22行 → 1790行
  ├─ 缺失值填补 (中位数)
  ├─ 异常值处理
  └─ 日期标准化
    ↓
[特征工程]
  ├─ 时间衍生: 季节, 周末, 节假日
  ├─ 活动衍生: 工作日类型, 经济活动期
  └─ 离散化: 污染物3分类
    ↓
[统计分析]
  ├─ 描述统计: 均值、标准差、分位数
  ├─ 相关分析: Pearson相关系数
  └─ 分布分析: 质量等级分布
    ↓
[关联规则挖掘]
  ├─ Apriori算法: 支持度≥5%
  ├─ 规则生成: 置信度≥50%
  └─ 规则筛选: 提升度>1.5 → 22,987条强规则
    ↓
[可视化与解释]
  ├─ 热力图: 相关性
  ├─ 时序图: 趋势
  ├─ 箱线图: 分布
  └─ 规则图: 模式
    ↓
治理建议与决策支持
```

### 6.2 核心算法

**Apriori算法核心步骤**:

```python
1. 扫描事务库，计算1-项集支持度
2. 递推k-项集:
   for k in range(2, n):
       候选集 = generate_candidates(频繁(k-1)-项集)
       for 候选项集c in 候选集:
           support(c) = count(c包含的事务) / 总事务数
           if support(c) ≥ minSupport:
               频繁k-项集.add(c)
3. 从频繁项集生成规则:
   for 频繁项集L:
       for 后件c ⊆ L:
          前件 = L - c
          confidence = support(L) / support(前件)
          lift = confidence / support(c)
          if confidence ≥ minConf AND lift ≥ minLift:
              规则库.add(规则)
```

**时间复杂度**: O(2^n) 其中n=项数
**空间优化**: 使用FP树可优化至O(n×depth)

### 6.3 关键参数说明

| 参数 | 含义 | 选择原则 |
|------|------|---------|
| Support | 项集出现频率 | 5-10% 平衡覆盖与精度 |
| Confidence | 规则准确度 | 50-80% 实用性要求 |
| Lift | 规则好度(提升度) | >1.5 统计显著性 |

### 7.4 局限性与改进方向

**现有局限性**:
1. 仅用中位数填补缺失值（可改用KNN）
2. 污染物3分类可能过粗（可用聚类）
3. 未考虑空间相关性（多个监测站）
4. 没有加入气象数据（温度、风速、湿度）

**改进方向**:
1. 融合气象数据，分离自然因素与人为排放
2. 采用时间序列模型（ARIMA/Prophet）预测
3. 使用FP-Growth替代Apriori提升性能
4. 加入因果推断方法验证规则因果性

## 结论

通过数据挖掘分析，我们成功揭示了北京空气质量的二阶特征：
1. **污染源识别**: 交通(NO₂)、工业(SO₂)、供暖(PM2.5)
